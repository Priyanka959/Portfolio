# Screenshots Directory

This directory contains screenshots of the portfolio website for documentation purposes.

## Required Screenshots

### 1. Home Page (`home.png`)
- Capture the landing page with hero section
- Include navigation bar and main content
- Show the full viewport on desktop

### 2. Projects Section (`projects.png`)
- Screenshot of the projects showcase
- Include project cards and descriptions
- Show hover effects if possible

### 3. Skills Section (`skills.png`)
- Technical skills display
- Proficiency indicators
- Technology stack visualization

### 4. Resume Section (`resume.png`)
- Resume preview iframe
- Download button
- PDF display functionality

### 5. Mobile View (`mobile.png`)
- Responsive design on mobile device
- Collapsed navigation menu
- Touch-friendly interface

### 6. Contact Section (`contact.png`)
- Contact form and information
- Social media links
- Footer content

## Screenshot Guidelines

- **Resolution**: 1920x1080 for desktop screenshots
- **Format**: PNG format for better quality
- **Browser**: Chrome/Firefox latest version
- **Device**: Include both desktop and mobile views
- **Naming**: Use descriptive, lowercase names with .png extension

## How to Take Screenshots

1. Start the development server: `npm start`
2. Open `http://localhost:4200` in your browser
3. Navigate to each section
4. Use browser dev tools to simulate mobile view for mobile screenshots
5. Take full-page screenshots
6. Save in this directory with appropriate names

## Tools for Screenshots

- Browser built-in screenshot tools
- Snipping Tool (Windows)
- Snagit
- Lightshot
- Browser extensions (Full Page Screen Capture)

## Notes

- Ensure all animations have loaded before taking screenshots
- Include hover states where relevant
- Test on multiple screen sizes
- Keep file sizes reasonable (< 2MB per image)
